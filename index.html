Dota 2 Clone

  • In this project, I recreated mechanics of Abilites system of Heros, Buffs and Debuffs and CoolDown system of Dota 2 Game.
  • Dota 2 is multiplayer online battle arena game, and all Heros of this game have unique abilities. I recreated abilities of few heroes and how their abilites affect other heroes.
  • All the information about the abilities are taken from wiki pages of this game.
  • Below are some examples of heroes I made, and I will continue to expllore and add more in future.


BRISTLEBACK :

  • BristleBack is melee tank hero and famous for his 2 PASSIVE abilites.
  • His first 2 abilites have low mana cost and low cooldown, and other 2 abilities are passive.
  • Below are the abilites of this hero.

First Ability -
  • Ability Type - Target Unit
  • Affects - Enemies
  • Discription - This ability add debuff to the target which reduce the armor and movement speed of target enemies. Multiple cast on same target stack up and refresh the duration of debuff.
  • Extra - This ability can stack upto maximum 4 times. The first debuff has more impact and after that stacked debuff affects the target less.
  • Source Code

Second Ability -
  • Ability Type - No Target
  • Affects - Enemies
  • Discription - This ability sprays the quills and deal damage to enemies around BristleBack. This ability also add debuff to affected targets.
  • Extra - Whenever affected targets again hit by Quill Spray, it deals Bonus damage till this Bonus damage cross the certain damage limit.
  • Source Code

Third Ability -
  • Ability Type - Passive
  • Affects - Self
  • Discription - This ability helps BristleBack to take less damage whenever he takes damage from rear or side.
  • Extra - If Bristleback take more than certain amount of damage from the back, he release Quill Spray.
  • Source Code

Forth Ability -
  • Ability Type - Passive
  • Affects - Self
  • Discription - This ability add Buff on BristleBack whenever he cast any ability and this buff increase the movement speed and damage of this hero.
  • Extra - This buff can stack upto maximum 5 times. Each stack has its own duration and does not reset the buff duration of previous stack.
  • Source Code

BRAID CLONE

  • In this project, I recreated mechanics of indie game called BRAID.
  • BRAID is a "2D" platformer puzzle game, where you can go back in time to solve various puzzles and complete level.
  • I recreated this reverse time mechanic, but in "3D" with 3rd person controls.
  • There are 2 modes in this project, one same as the game and other I created just for fun.

The main difference between these 2 modes are :
Difference Mode - 1 Mode - 2
Storage Time Unlimited Fixed
Reverse Time Rate Normal Fast-Paced
Control over Ability Yes No
Control Condition Till toggle off or First position reached Till first position reached


MODE - 1 :

  • In this mode, I keep the original mechanics of time reverse travel.
  • I stored data from start of the game to the UNLIMITED time till the ability is activated.
  • When ability is activated, player and moving platform move back to their privious position one by one at NORMAL RATE till ability is toggled off.
  • After the first time, the data will be stored from after the last time ability toggle OFF to the UNLIMITED time till when again ability is activated.
  • Source Code; UPPER SECTION
  • Below are some examples of this mode.

White Lines shows the stored positions of Objects, and the length of the lines are bigger than Mode - 2 lines, which indicates UNLIMITED storage capacity of Mode - 1.

MODE - 2 :

  • In this mode, I apply some changes to the original mechanics of time reverse travel.
  • I stored data from start of the game to the FIXED time till the ability is activated.
  • When ability is activated, player and moving platform move back to their privious position one by one at MORE SPEED till the first position is reached.
  • After the first time, the data will be stored from after the last time ability completed to till the next FIXED time when again ability is activated.
  • Source Code; LOWER SECTION
  • Below are some examples of this mode.

White Lines shows the stored positions of Objects, and the length of the lines are shorter and limited than Mode - 1 lines, which indicates storage capacity of Mode - 2.
